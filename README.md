---

## О проекте (Краткое содержание)

**Проблема:** Клиенты используют банковские приложения исключительно утилитарно (только для переводов), игнорируют рекламные баннеры сложных продуктов и обладают низким уровнем финансовой грамотности и кибергигиены.

**Решение:** **«Уголок благополучия»** — это встраиваемая платформа, которая превращает скучное изучение финансов в интерактивный симулятор. Мы не просто развлекаем, а **обучаем через действие**, создавая естественный контекст для продажи банковских продуктов.

---

## Ценностное предложение (Business Value)

Наш продукт решает 3 ключевые задачи розничного банка:

### 1. Рост перекрестных продаж (Cross-sell)
Вместо прямой навязчивой рекламы мы используем **«Умные призывы к действию»**.
*   **Механика:** Пользователь сначала взаимодействует с виртуальным аналогом продукта (например, «Облигации») в игре, понимает принцип его работы и доходность.
*   **Результат:** Снижается барьер страха и непонимания. Предложение открыть реальный счет поступает в момент максимального интереса (контекстная продажа).

### 2. Снижение рисков и социальная ответственность
*   **Механика:** Интерактивные сценарии **«Неожиданный звонок»**. Пользователь проживает атаку мошенников в безопасной среде симулятора.
*   **Результат:** Повышение финансовой грамотности клиентов, снижение нагрузки на службу поддержки и укрепление имиджа банка как «заботливого партнера».

### 3. Удержание и лояльность клиентов
*   **Механика:** Накопительный цикл (получение дохода -> реинвестирование) и экономика поощрений.
*   **Результат:** Формирование привычки ежедневного входа в приложение, увеличение длительности сессии и эмоциональной привязанности к экосистеме банка.

## Логика продукта

Мы отошли от устаревшей модели «наград за простые нажатия». Наш подход — **Полноценная симуляция**.

| Элемент игры | Банковская метафора | Цель для бизнеса |
| :--- | :--- | :--- |
| **Покупка Актива** | Открытие Вклада/Счета | Знакомство с продуктовой линейкой |
| **Улучшение** | Реинвестирование % | Обучение финансовой дисциплине |
| **Событие (Звонок)** | Атака мошенников | Обучение безопасности + Продажа страховки |
| **Валюта (Алмазы)** | Баллы лояльности | Мотивация на целевые действия (оформление карты и т.д.) |

---

## Технологии и интеграция

Решение спроектировано для **быстрого вывода на рынок** без утяжеления основного приложения банка.

*   **Платформа:** HTML5 (Одностраничное приложение).
*   **Интеграция:** Бесшовное встраивание через системный веб-компонент (WebView) в приложения на iOS/Android.
*   **Архитектура:** Модульная, событийно-ориентированная. Позволяет удаленно настраивать баланс, тексты и сценарии без необходимости обновлять приложение в магазинах.
*   **Производительность:** Использование процедурных анимаций обеспечивает минимальный вес загружаемых файлов и плавную работу на любых устройствах.

## План развития (Потенциал для пилота)

1.  **Этап 1 (Текущий):** Базовая версия (MVP) с основным игровым циклом, сценарием кибербезопасности и системой контекстных предложений.
2.  **Этап 2 (Пилот):** Интеграция с реальным API банка для отслеживания переходов по ссылкам и начисления наград за реальные действия (например, открытие вклада).
3.  **Этап 3 (Масштабирование):** Трансформация в единый **Центр Игрофикации**, объединяющий все маркетинговые активности банка в единую мета-вселенную.
